// ABOUT VERSIONING

"express": "^5.2.1" - framework: version

3 main parts
1st: 5
2nd: 2
3rd: 1

3rd part:
Minor Fixes
Optional Updates
eg-5.2.3

2nd part:
Recommended Bug Fix 
(Mostly Security Fixes)
eg-5.3.1

1st part:
Major releases 
Breaking Updates
eg-6.3.1

// Making installion specific versions
npm install express -> gets the latest stable version

npm install express@4.17.2 -> gets the 4.17.2 version of express 

// what does '^' mean
"express": "^5.2.1" -> lock version 5
ie, install 5.2.1 to <6.0.0
(compatable version)

// what does '~' mean
"express": "~5.2.1" -> only upgrade minor chages ie 3rd number

~5.2.1 ok
~5.2.2 ok
~5.2.3 ok

~5.3.0 NOT OK
